# 공모주 알리미 앱 - 종합 기획서

> **앱 컨셉**: "오늘 내가 해야 할 일"이 바로 보이는 공모주 생애주기 관리앱
> **핵심 가치**: 정보 제공이 아닌, 관심 → 청약 → 환불 → 상장 → 매도 → 수익까지 끝까지 관리

---

## 1. PRD (Product Requirements Document)

### 1.1 제품 개요

| 항목 | 내용 |
|------|------|
| 제품명 | 공모주 알리미 (가칭: IPO Keeper) |
| 플랫폼 | iOS / Android |
| 타겟 유저 | 대한민국 개인 공모주 투자자 (20~50대) |
| 핵심 가치 | 청약 생애주기 끝까지 관리 + 촘촘한 개인화 알림 |

### 1.2 문제 정의

- 기존 앱들은 일정 조회에 치중, 청약 후 관리 없음
- 환불일/상장일을 직접 캘린더에 기록해야 함
- 수익 기록이 없어 누적 성과 파악 불가
- 최소 청약 금액을 직접 계산해야 함
- 알림이 단발적이고 맥락이 없음

### 1.3 핵심 기능 요구사항

#### F1. 공모주 일정 조회
- 종목명, 수요예측 일정, 청약 일정, 환불일, 상장일
- 공모가 밴드 / 확정 공모가
- 주간사 목록
- 경쟁률 (실시간 업데이트)
- 최소 청약 수량 / **최소 청약 가능 금액 (자동 계산 표시)**
- 기업 요약 (업종, 시가총액, 사업 내용)

#### F2. 즐겨찾기
- 관심 공모주 저장/해제
- 청약 시작 전날, 당일 오전, 마감일 알림

#### F3. 청약 기록
- 증권사, 청약 수량, 납입 증거금
- 배정 수량, 환불 금액
- 매도 가격, 매도 수량, 수익 금액
- 상태값 자동 관리 (청약완료 → 배정확인 → 환불완료 → 상장 → 매도완료)

#### F4. 개인화 알림
- 관심종목: D-3, D-1, 청약당일, 청약마감일
- 청약완료 종목: 환불일 전날, 환불당일
- 상장일 전날, 상장당일 오전 8시
- 매도 후: 입금 확인 알림, 이체 권유 알림

#### F5. 위젯
- 오늘의 공모주 액션 요약 (소형)
- 이번 주 공모주 캘린더 (중형)
- 내가 청약한 종목 상태 (중형)

#### F6. 수익 관리
- 누적 수익, 월별 수익, 종목별 수익
- 참여 횟수, 평균 수익, 수익률

### 1.4 비기능 요구사항

- 오프라인에서도 저장된 데이터 조회 가능
- 알림 정확도: 설정 시간 ±5분 이내
- 앱 초기 로딩 2초 이내
- 공모주 데이터 최소 1일 1회 갱신

## 3. 사용자 플로우

### 3.1 신규 유저 온보딩 플로우

```
앱 실행
  → 스플래시 (공모주 브랜드 소개)
  → 알림 권한 요청
  → 홈 진입
```

### 3.2 공모주 탐색 → 즐겨찾기 플로우

```
홈 / 공모주 리스트
  → 종목 탭
  → 상세 화면 확인 (일정, 공모가, 기업 요약)
  → ★ 즐겨찾기 등록
  → 알림 설정 선택 (청약 D-1, 당일, 마감일)
  → 완료
```

### 3.3 청약 기록 플로우

```
공모주 상세 or 내 청약 탭
  → "청약 기록 추가"
  → 증권사 선택
  → 청약 수량 입력
  → 납입 증거금 자동 계산 표시 (수정 가능)
  → 저장
  → 상태: 청약완료
  → (배정 발표 후) 배정 수량 / 환불 금액 입력
  → (상장 후) 매도 가격 / 매도 수량 / 수익 입력
  → 상태: 매도완료 → 수익 통계 반영
```

### 3.4 알림 수신 플로우

```
푸시 알림 수신
  → 탭하여 앱 실행
  → 해당 종목 상세 또는 내 청약 화면으로 딥링크
  → 필요한 액션 수행 (기록 입력 등)
```

## 5. 화면 목록 및 핵심 UI 요소

### 5.1 홈 (Home)

**목적**: 오늘 해야 할 일을 한눈에

| UI 요소 | 설명 |
|---------|------|
| 상단 날짜/인사 | "오늘은 4월 7일, 해야 할 일이 2건 있어요" |
| 액션 카드 리스트 | 오늘 청약 마감, 환불일, 상장일 종목 카드 |
| 각 카드 | 종목명, 상태 배지, CTA 버튼 ("기록하기", "확인") |
| 이번 주 미리보기 | 가로 스크롤 달력 (7일) + 이벤트 도트 |
| 관심 종목 섹션 | 즐겨찾기 종목 상태 카드 |

### 5.2 공모주 리스트 (IPO List)

**목적**: 전체 공모주 탐색

| UI 요소 | 설명 |
|---------|------|
| 상단 탭 | 전체 / 청약중 / 청약예정 / 상장예정 |
| 종목 카드 | 종목명, D-day 배지, 공모가, 최소청약금, 즐겨찾기 버튼 |
| D-day 배지 | 청약까지 D-3, 청약중, 상장 D-1 등 컬러 구분 |
| 정렬/필터 | 상장일순, 청약일순, 공모가순 |

### 5.3 공모주 상세 (IPO Detail)

**목적**: 종목 정보 + 청약 결정

| UI 요소 | 설명 |
|---------|------|
| 헤더 | 종목명, 업종 태그, 즐겨찾기 버튼 |
| 일정 타임라인 | 수요예측 → 청약 → 환불 → 상장 단계별 표시 |
| 공모가 섹션 | 밴드 하단~상단, 확정가, 시가총액 |
| 청약 정보 | 최소 청약 수량, **최소 청약 금액 (굵게 강조)**, 주간사 |
| 수요예측 결과 | 경쟁률, 의무보유 확약 비율 |
| 기업 요약 | 사업 내용 2~3줄, 업종, 매출 |
| 하단 CTA | "즐겨찾기 추가" / "청약 기록하기" 버튼 |

### 5.4 내 청약 (My IPO)

**목적**: 청약한 종목의 생애주기 관리

| UI 요소 | 설명 |
|---------|------|
| 상단 탭 | 진행중 / 완료 |
| 종목 카드 | 상태 배지, 종목명, 내 청약 수량, 다음 액션 안내 |
| 상태 배지 | 청약완료(파랑), 배정확인(노랑), 환불완료(초록), 상장대기(보라), 매도완료(회색) |
| 다음 액션 | "배정 수량 입력하기", "환불금 확인하기" 등 |
| 상세 입력 | 슬라이딩 시트로 각 단계 데이터 입력 |
| FAB | "청약 추가" 플로팅 버튼 |

### 5.5 캘린더 (Calendar)

**목적**: 전체 공모주 일정 한눈에 보기

| UI 요소 | 설명 |
|---------|------|
| 월간 달력 | 이벤트 타입별 컬러 도트 (청약/환불/상장) |
| 날짜 탭 | 선택 날짜의 이벤트 리스트 |
| 이벤트 카드 | 종목명, 이벤트 타입, 내 청약 여부 표시 |
| 범례 | 하단 컬러 범례 (수요예측/청약/환불/상장) |

### 5.6 수익 대시보드 (Profit)

**목적**: 공모주 투자 성과 확인

| UI 요소 | 설명 |
|---------|------|
| 상단 요약 카드 | 누적 수익, 참여 횟수, 평균 수익 |
| 월별 바 차트 | 최근 6개월 수익 |
| 종목별 리스트 | 종목명, 수익, 수익률, 날짜 |
| 통계 섹션 | 최고 수익 종목, 평균 수익률 |

### 5.7 설정 (Settings)

| UI 요소 | 설명 |
|---------|------|
| 알림 토글 | 청약일/환불일/상장일 알림 on/off |
| 알림 시간 설정 | 각 이벤트별 알림 시간 설정 (예: 오전 8시) |
| 데이터 초기화 | 내 청약 기록 삭제 |
| 앱 버전 / 문의 | |

## 6. DB 스키마 초안

### 6.1 IPO (공모주 기본 정보)

```sql
CREATE TABLE ipo (
  id                    TEXT PRIMARY KEY,        -- 고유 ID (예: "2025-삼성바이오")
  company_name          TEXT NOT NULL,           -- 종목명
  ticker                TEXT,                    -- 종목코드 (상장 후)
  sector                TEXT,                    -- 업종
  business_summary      TEXT,                    -- 기업 요약
  
  -- 일정
  demand_forecast_start DATE,                    -- 수요예측 시작
  demand_forecast_end   DATE,                    -- 수요예측 종료
  subscription_start    DATE,                    -- 청약 시작
  subscription_end      DATE,                    -- 청약 마감
  refund_date           DATE,                    -- 환불일
  listing_date          DATE,                    -- 상장일
  
  -- 공모가
  price_band_low        INTEGER,                 -- 공모가 밴드 하단
  price_band_high       INTEGER,                 -- 공모가 밴드 상단
  confirmed_price       INTEGER,                 -- 확정 공모가
  
  -- 청약 정보
  total_shares          INTEGER,                 -- 총 공모 주식수
  min_subscription_qty  INTEGER,                 -- 최소 청약 수량
  subscription_ratio    REAL,                    -- 증거금 비율 (예: 0.5 = 50%)
  
  -- 경쟁률
  competition_rate      REAL,                    -- 경쟁률
  
  -- 주간사 (JSON 배열)
  lead_underwriters     TEXT,                    -- JSON: ["미래에셋", "NH투자"]
  
  -- 메타
  status                TEXT DEFAULT 'upcoming', -- upcoming/subscription/listing/closed
  created_at            DATETIME,
  updated_at            DATETIME
);
```

### 6.2 Watchlist (즐겨찾기)

```sql
CREATE TABLE watchlist (
  id         INTEGER PRIMARY KEY AUTOINCREMENT,
  ipo_id     TEXT NOT NULL REFERENCES ipo(id),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  
  -- 알림 설정
  notify_d3  INTEGER DEFAULT 1,   -- 청약 D-3 알림
  notify_d1  INTEGER DEFAULT 1,   -- 청약 D-1 알림
  notify_day INTEGER DEFAULT 1,   -- 청약 당일 알림
  notify_end INTEGER DEFAULT 1    -- 청약 마감일 알림
);
```

### 6.3 Subscription (청약 기록)

```sql
CREATE TABLE subscription (
  id                   INTEGER PRIMARY KEY AUTOINCREMENT,
  ipo_id               TEXT NOT NULL REFERENCES ipo(id),
  
  -- 청약 단계
  broker               TEXT,                    -- 증권사
  applied_qty          INTEGER,                 -- 청약 수량
  deposit_amount       INTEGER,                 -- 납입 증거금
  
  -- 배정 단계
  allocated_qty        INTEGER,                 -- 배정 수량
  refund_amount        INTEGER,                 -- 환불 금액
  
  -- 매도 단계
  sell_price           INTEGER,                 -- 매도 가격
  sell_qty             INTEGER,                 -- 매도 수량
  profit_amount        INTEGER,                 -- 수익 금액 (세전)
  profit_rate          REAL,                    -- 수익률
  
  -- 상태
  status               TEXT DEFAULT 'applied',
  -- applied: 청약완료
  -- allocated: 배정확인
  -- refunded: 환불완료
  -- listed: 상장
  -- sold: 매도완료
  
  -- 메모
  memo                 TEXT,
  
  created_at           DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at           DATETIME
);
```

### 6.4 Notification Log (알림 이력)

```sql
CREATE TABLE notification_log (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  ipo_id       TEXT,
  type         TEXT,                -- 알림 타입
  title        TEXT,
  body         TEXT,
  sent_at      DATETIME,
  opened       INTEGER DEFAULT 0    -- 열람 여부
);
```

## 7. 상태값 설계

### 7.1 공모주 (IPO) 상태

```
upcoming        수요예측 전
forecasting     수요예측 중
subscribing     청약 중
waiting_listing 청약 마감 ~ 상장 전
listed          상장 완료
closed          종료
```

### 7.2 내 청약 (Subscription) 상태

```
applied     → 청약 완료 (수량, 증거금 입력됨)
allocated   → 배정 확인 (배정 수량 입력됨)
refunded    → 환불 완료 (환불 금액 입력됨)
listed      → 상장됨 (매도 대기)
sold        → 매도 완료 (수익 기록됨)
```

### 7.3 상태 전이도

```
[applied]
    ↓ 배정 발표 후 수량 입력
[allocated]
    ↓ 환불일 처리 후 금액 입력
[refunded]
    ↓ 상장일 자동 전이 or 수동 확인
[listed]
    ↓ 매도 가격/수량/수익 입력
[sold]
```

### 11.2 IPO 모델

```dart
// lib/data/models/ipo.dart

class IPO {
  final String id;
  final String companyName;
  final String sector;
  final String businessSummary;
  final DateTime? subscriptionStart;
  final DateTime? subscriptionEnd;
  final DateTime? refundDate;
  final DateTime? listingDate;
  final int? priceBandLow;
  final int? priceBandHigh;
  final int? confirmedPrice;
  final int minSubscriptionQty;
  final double subscriptionRatio;
  final List<String> leadUnderwriters;
  final double? competitionRate;
  final IPOStatus status;

  // 계산 필드
  int? get minSubscriptionAmount {
    if (confirmedPrice == null) return null;
    return (confirmedPrice! * minSubscriptionQty * subscriptionRatio).toInt();
  }

  String get minSubscriptionAmountFormatted {
    final amount = minSubscriptionAmount;
    if (amount == null) return '확정 전';
    if (amount >= 10000) {
      return '${(amount / 10000).toStringAsFixed(1)}만원';
    }
    return '${amount.toString().replaceAllMapped(RegExp(r'(\d{1,3})(?=(\d{3})+(?!\d))'), (m) => '${m[1]},')}원';
  }
}

enum IPOStatus { upcoming, forecasting, subscribing, waitingListing, listed, closed }
```

### 11.3 홈 화면 액션 카드 로직

```dart
// lib/domain/usecases/get_today_actions.dart

class TodayAction {
  final IPO ipo;
  final ActionType type;
  final String message;
  final String ctaLabel;
}

enum ActionType {
  subscriptionStart,    // 오늘 청약 시작
  subscriptionEnd,      // 오늘 청약 마감
  refund,              // 오늘 환불일
  listing,             // 오늘 상장일
}

List<TodayAction> getTodayActions(List<IPO> ipos, DateTime today) {
  final actions = <TodayAction>[];
  for (final ipo in ipos) {
    final sub = ipo.subscriptionStart;
    final end = ipo.subscriptionEnd;
    final ref = ipo.refundDate;
    final lst = ipo.listingDate;

    if (sub != null && isSameDay(sub, today))
      actions.add(TodayAction(ipo: ipo, type: ActionType.subscriptionStart, ...));
    if (end != null && isSameDay(end, today))
      actions.add(TodayAction(ipo: ipo, type: ActionType.subscriptionEnd, ...));
    if (ref != null && isSameDay(ref, today))
      actions.add(TodayAction(ipo: ipo, type: ActionType.refund, ...));
    if (lst != null && isSameDay(lst, today))
      actions.add(TodayAction(ipo: ipo, type: ActionType.listing, ...));
  }
  return actions;
}
```

## 12. 데이터 수집 방식 (현실적 제안)

### 12.1 방식 비교

| 방식 | 장점 | 단점 | MVP 적합도 |
|------|------|------|-----------|
| 공식 API (한국거래소, DART) | 정확, 자동화 | 공모주 상세 데이터 부족 | 부분 활용 |
| 크롤링 (38커뮤니케이션, IPO스탁 등) | 풍부한 데이터 | 법적 리스크, 유지보수 | 리서치 보조 |
| **관리자 직접 입력** | 정확도 최고, 커스텀 | 인력 필요 | **MVP 1순위** |
| 반자동 (크롤링 → 검토 후 등록) | 속도+정확도 균형 | 개발 비용 | Phase 2 |

### 12.2 MVP 데이터 수집 운영 방법

```
[추천 프로세스]
1. Supabase Studio를 관리자 패널로 활용
2. 공모 일정 발표 시 담당자가 직접 입력 (1건당 5분 이내)
3. 확정 공모가 발표 시 해당 필드 업데이트
4. 경쟁률은 청약 기간 중 1~2회 수동 업데이트

[참고 데이터 소스]
- 금융감독원 DART (dart.fss.or.kr) - 증권신고서
- 한국거래소 (ipo.krx.co.kr) - 공모 일정
- 38커뮤니케이션 (38.co.kr) - 경쟁률, 청약 정보
```

### 12.3 데이터 갱신 주기

| 데이터 | 갱신 주기 | 방법 |
|--------|-----------|------|
| 공모 일정, 기업 정보 | 발표 즉시 | 관리자 입력 |
| 확정 공모가 | 수요예측 완료 후 | 관리자 업데이트 |
