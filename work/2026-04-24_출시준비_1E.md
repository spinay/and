## 주요 변경 내용

### 1. 도메인 레이어 추출 + 테스트

- `lib/domain/today_actions.dart` — 홈 "오늘 할 일" 순수 함수 (`computeTodayActions`)
- `lib/domain/profit_summary.dart` — 수익 요약 계산 (`ProfitSummary.from`)
- `lib/domain/ipo_list_sorting.dart` — D-day 정렬·필터 enum

Home/Profit/IPOList 화면을 이 순수 함수를 쓰도록 교체해 비즈니스 로직을 테스트 가능하게 만들었다.

### 2. 알림 시스템 안정화

**문제**: 기존엔 관심종목 토글 시 알림이 앱 재시작 때만 반영됐고,
Android 13+ POST_NOTIFICATIONS 런타임 권한 요청이 없었으며,
설정 화면 토글은 UI만 있고 실제로는 아무 일도 하지 않았다.

**해결**:
- `IpoNotificationScheduler` 인터페이스 도입 → 테스트에서 `NoopNotificationScheduler` 주입 가능
- `WatchlistRepository`가 이 인터페이스에 의존 → 토글 시 즉시 스케줄/취소
- `NotificationService.requestPermissions()` 추가 (관심 등록 시 자동 호출)
- `NotificationPrefs` (SharedPreferences) + 설정 화면 토글 실동작
- 설정 토글 변경 시 `NotificationService.applyPrefs()` → `cancelAll` → 전체 재스케줄

### 3. 상세 화면 CTA 연결

`ipo_detail_screen.dart`의 "청약 기록하기" 버튼이 "기능 준비 중" SnackBar만 띄우던 것을 `/subscription/new?ipoId=...` 라우트로 실제 연결.

### 4. 설정 화면 완성

- 3종 알림 토글: 청약 D-1 / 환불일 / 상장일 (SharedPreferences 저장)
- 청약 기록 초기화: 실제 DB 전체 삭제
- 관심종목 모두 해제: 실제 해제 + 예약 알림까지 취소
- 홈 화면 우상단에서 진입 가능 (⚙ 아이콘 추가)

### 5. 앱 아이덴티티

- Android label: `ipo_keeper` → `공모주 알리미`
- iOS `CFBundleDisplayName` / `CFBundleName`: `Ipo Keeper` → `공모주 알리미`

## 출시 전 남은 일 (이 세션 바깥)

이 세션에서 해결할 수 없는, **운영자 권한이 필요한** 항목:

1. **Android release 서명 키스토어 생성**
   - 현재 `android/app/build.gradle.kts`의 release 빌드는 debug key를 사용 중
   - Play Store 업로드 전에 `keytool -genkey`로 upload key 만들고 `signingConfigs { release { ... } }` 추가 필요
2. **앱 아이콘 교체**
   - 현재 기본 Flutter 아이콘(`@mipmap/ic_launcher`). `flutter_launcher_icons` 패키지로 교체 권장
3. **Play Console 등록용 자료**
   - 1024x1024 마스터 아이콘
   - 피처 그래픽(1024x500), 스크린샷 최소 2장
   - 개인정보 처리방침 URL (로컬 DB만 쓰지만 Play Console은 필수)
4. **iOS 개발자 계정 배포 프로필**
   - Team ID 설정, `CFBundleIdentifier` 확정

이상의 외부 권한·자산이 갖춰지면 위 APK와 동일한 코드로 바로 스토어 업로드 가능.
