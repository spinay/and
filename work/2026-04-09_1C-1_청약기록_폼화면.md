## 상세 내용

### 1. 청약 기록 추가/수정 폼 화면

`lib/presentation/subscription_form/subscription_form_screen.dart` 신규 작성.

**기능:**
- **신규 모드**: 종목 선택 (BottomSheet IPO 목록) → 증권사, 수량, 증거금 입력
- **수정 모드**: 기존 Subscription 불러와서 편집 + 삭제 가능
- **자동 계산**:
  - 종목 선택 시 최소 청약 수량 자동 입력
  - 청약 수량 변경 시 증거금 자동 계산 (`수량 × 공모가 × 증거금비율`)
  - 배정 수량 입력 시 환불 금액 자동 계산 (`증거금 - 배정수량 × 공모가`)
  - 매도 가격/수량 입력 시 수익 미리보기 표시
- **진행 상태 선택**: ChoiceChip으로 applied/allocated/refunded/listed/sold 선택
- **섹션 자동 펼침**: 상태에 따라 기본정보 → 배정단계 → 매도단계가 점진적으로 펼쳐짐
- **initialFocus 지원**: home 카드 CTA에서 `allocation`/`sale`로 진입 시 해당 섹션 바로 펼침 + 저장 시 상태 자동 승격

**설계 원칙:**
- catalog(IPO 목록)은 운영자가 Google Sheet → JSON으로 관리 (CatalogRepository)
- 청약 기록은 사용자 개인 데이터 → Drift DB로 로컬 영속화 (SubscriptionRepository)
- 두 쪽은 `canonical_key`로만 연결, 분리 독립

### 2. Router 등록

`lib/core/router/app_router.dart` 수정:
- `/subscription/new` — 신규 추가. `?ipoId=...` 쿼리로 종목 프리픽스 가능
- `/subscription/edit/:id` — 수정. `?focus=allocation|sale` 쿼리로 진입 섹션 지정

### 3. my_ipo FAB + 카드 탭 연결

`lib/presentation/my_ipo/my_ipo_screen.dart` 수정:
- FAB: `SnackBar("준비 중")` → `context.push('/subscription/new')`
- `_SubscriptionCard`: `GestureDetector` 감싸서 탭 → `/subscription/edit/$id`

### 4. 남은 작업 (home CTA 연결)

home 화면의 오늘 할 일 카드 CTA가 현재 상세 화면(`/detail/:id`)으로만 이동. 다음 단계:
- "환불금 기록하기" → `/subscription/edit/:id?focus=allocation`
- "매도 기록하기" → `/subscription/edit/:id?focus=sale`
- "청약하러 가기" → `/subscription/new?ipoId=...`
