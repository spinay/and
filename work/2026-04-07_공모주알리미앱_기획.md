## 핵심 결정사항

### 앱 핵심 차별점
- 공모주 정보 제공이 아닌 **청약 생애주기 끝까지 관리**
- 관심 → 청약 → 환불 → 상장 → 매도 → 수익까지 상태 관리
- 최소 청약 가능 금액 자동 계산 표시

### 기술 스택
- **Flutter** (iOS/Android 동시, 위젯 지원)
- **Riverpod** 상태관리
- **Drift** 로컬 SQLite
- **Supabase** 백엔드 (MVP 빠른 세팅)

### 데이터 수집 (MVP)
- 관리자 직접 입력 (Supabase Studio)
- 데이터 소스: DART, 한국거래소, 38커뮤니케이션

### 개발 일정
- Week 1-2: 기반 + 공모주 리스트
- Week 3-4: 청약 기록 + 상태관리
- Week 5-6: 알림 + 홈 화면
- Week 7-8: 수익 대시보드 + 출시 준비
